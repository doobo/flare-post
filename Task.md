# 「Cloud Web Offers」首页优化方案

> 文档版本：V2.0
> 适用场景：优惠聚合/云计算优惠信息展示首页
> 优化目标：提升信息传达效率、优化视觉层级、增强用户决策引导

---

## 一、当前页面问题诊断

| 问题 | 具体表现 |
|------|----------|
| 头部空间浪费 | Hero 区域占首屏近 1/2，标题 + 副标题 + 搜索框垂直堆叠，搜索框以下内容首屏不可见 |
| 搜索功能弱 | 搜索仅触发 API 请求，无防抖、无搜索高亮、无搜索建议/历史，体验粗糙 |
| 缺少分类筛选 | 卡片只展示 `post.category` 标签，无顶部分类导航/筛选入口，用户无法按类别浏览 |
| 时间信息冗余 | 显示 `created_at` 发布日期，用户更关心"剩余天数"而非"哪天发布" |
| CTA 交互弱 | "Read details" 仅为文字链接加箭头图标，缺少按钮感，点击欲望低 |
| 品牌感缺失 | 页面整体简洁但无记忆点，缺少品牌色系统、图标体系、视觉锚点 |
| 无社会证明 | 缺少评分、浏览/购买人数、推荐标签等信任元素，用户决策成本高 |
| 加载体验差 | 仅有简单 spinner，无骨架屏，白屏时间长；首次请求全部数据无分页 |
| 卡片信息密度低 | 卡片仅展示分类、标题、摘要、日期，缺少价格/折扣/有效期等关键决策信息 |
| 无排序能力 | 列表按 API 返回顺序排列，用户无法按热度、价格、剩余时间排序 |
| 空状态引导弱 | 搜索无结果时仅显示"没有找到"，无推荐替代或引导用户调整关键词 |
| 缺少页面元信息 | 无 `<title>` 动态更新、无 SEO meta 描述，不利于搜索引擎收录 |

## 二、优化方案

### 1. 布局结构优化

```
┌──────────────────────────────────────┐
│  Hero 精简区域                        │
│  [Logo] Cloud Web Offers    [CTA]     │
│  搜索框 (max-w-xl, 居中)              │
├──────────────────────────────────────┤
│  分类导航 (水平标签栏)                 │
│  全部 | 虚拟机 | VPN | 域名 | 其他    │
├──────────────────────────────────────┤
│  排序 & 统计  [最新] [即将过期] [热度] │  共 N 个优惠
├──────────────────────────────────────┤
│  卡片列表 (Grid 2/3 列)              │
│  ┌──────┐ ┌──────┐ ┌──────┐        │
│  │Card 1│ │Card 2│ │Card 3│        │
│  └──────┘ └──────┘ └──────┘        │
│  ...                                │
├──────────────────────────────────────┤
│  分页 / 加载更多                      │
└──────────────────────────────────────┘
```

### 2. Hero 区域改造

- **高度压缩**：从当前 300px+ 减少至 180-200px，释放更多首屏空间给内容列表
- **搜索框增强**：添加防抖（300ms）、搜索高亮（匹配文字加粗/变色）、为空时显示热门搜索词
- **品牌标识**：添加简化 Logo 图标（纯 CSS/SVG），强化品牌识别

### 3. 分类导航（新增）

- **位置**：Hero 下方，水平标签栏
- **数据来源**：从 posts 数据中动态提取所有 `category` 去重，或新增 API `/api/categories`
- **交互**：点击切换 category 过滤，选中态高亮（indigo 下划线/填充），支持"全部"重置

### 4. 卡片内容优化

**当前卡片结构：**
```
[Category 标签]                   [日期]
[标题 (2行)]                    
[摘要预览 (3行)]                
[Read details >]
```

**优化后卡片结构：**
```
[🔥 限时] [Category 标签]         [剩余 7 天]
[标题 (2行)]
[摘要预览 (2行)]
[折扣/价格标签] 💰 50% OFF
[👍 234 人已购]          [Read details →]
```

- **替换日期**：发布日期 → 截止日期倒计时（"剩 X 天"），过期显示"已结束"
- **新增折扣标签**：从 frontmatter 提取 `discount` 字段，卡片顶部视觉化展示
- **新增社会证明**：如有点击/购买数据，展示"N 人已购"/"N 次浏览"
- **CTA 按钮化**："Read details" 改为实心按钮或更大点击区域，配合 hover 动效

### 5. 新增排序 & 搜索高亮

- **排序下拉**：新增时间倒序、剩余时间升序（即将过期优先）、热度（需后端支持）
- **搜索高亮**：`searchQuery` 匹配文本在标题和摘要中包裹 `<mark>` 标签渲染

### 6. 加载体验

- **骨架屏**（Skeleton）：列表加载时显示 4-6 个灰色卡片骨架动画，替代 spinner
- **分页/无限加载**：后端新增 `?page=&limit=` 参数，前端滚动加载或"Load More"按钮
- **图片懒加载**：如后续卡片增加图片，使用 `loading="lazy"`

### 7. 空状态 & 错误处理

- **搜索无结果**：展示更友好的空状态（搜索图标 + "没有找到相关优惠" + "试试其他关键词" + 热门推荐）
- **网络错误**：捕获 fetch 异常，展示重试按钮，而非静默失败
