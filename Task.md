# 图片管理

## 1. 图片上传配置管理

新增 **图片上传配置表**，用于统一管理图片存储服务配置。

### 支持字段

- 配置名称
- 是否默认配置（全局唯一）
- 是否启用代理访问
- 存储类型（S3、R2、common 等）
- 上传接口地址
- Access Key
- Secret Key
- Refresh Token（如需要）
- 状态（启用/禁用）
- 排序号
- 备注信息

### 功能要求

- 支持新增、编辑、删除配置
- 支持设置默认配置
- 系统上传图片时优先使用默认配置

---

## 2. 默认图片存储配置

系统初始化时默认内置一套 **common** 配置：

- 默认设为启用状态
- 默认开启代理访问
- 作为系统默认上传配置

### Worker 图片代理能力

访问格式：

```text
/img/{id}.jpg
/img/{id}.png
```

处理逻辑：

1. 根据文件 ID 查询文件记录
2. 获取对应原始图片地址
3. Worker 转发请求并返回图片内容
4. 隐藏真实图片源站地址
5. 降低第三方图床链接被屏蔽或拦截的风险

- 按配置筛选

- 查看代理地址
- 删除文件记录

## 4. Offer 图片上传流程

Offer 发布时的图片上传流程调整如下：

```text
选择图片
    ↓
获取默认图片配置
    ↓
上传至对应存储服务
    ↓
获得原始 URL
    ↓
写入文件管理表
    ↓
判断是否开启代理
        ↓
是 → 返回代理 URL
        ↓
否 → 返回原始 URL
```

### 具体规则

- 上传时自动匹配系统默认配置
- 上传成功后同步创建文件管理记录
- 文件记录保存原始 URL 与代理 URL
- 如果配置开启代理：
    - 前端返回代理 URL
- 如果配置未开启代理：
    - 前端返回原始 URL

## 5. 图片不存在处理

当通过代理地址访问图片时：

```text
/img/{id}.jpg
```

处理逻辑：

1. 查询文件 ID
2. 文件存在：
    - 返回真实图片内容
3. 文件不存在：
    - 返回默认占位图

### 新增静态资源

```text
/public/images/not-found.png
```

### 默认占位图设计要求

- 简洁风格
- 显示「Image Not Found」
- 支持浅色和深色背景
- 用于图片资源不存在、已删除或异常场景

### 返回示例

```text
/img/123.jpg      → 正常图片
/img/999.jpg      → not-found.png
```
