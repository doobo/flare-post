# 后台Publish页面优化方案
## 一、布局与结构优化

### 1.1 左右分栏 + 独立滚动
- 左侧为表单填写区，右侧为实时预览区
- 左右两侧独立滚动，互不影响
- 右侧预览区固定显示当前最新预览内容

### 1.2 顶部步骤引导
新增步骤条，将流程拆分为三个清晰阶段：

| 步骤 | 名称 | 说明 |
|------|------|------|
| 1 | 基本信息 | 填写标题、分类、标签 |
| 2 | 优惠详情 | 编辑 Markdown 内容 |
| 3 | 预览发布 | 确认预览并发布 |

### 1.3 页面标题优化
- 原标题：`Publish New Offer`
- 优化后：`发布新优惠` + 副标题说明
- 副标题示例：*填写以下信息，即可创建并发布新的优惠活动，吸引更多用户关注。*

---

## 二、表单字段优化

### 2.1 标题（Title）

| 优化点 | 说明                              |
|--------|---------------------------------|
| 字数限制 | 不超过 90 个字符，实时显示剩余字数             |
| 占位符示例 | 例如：`50% Off VPS Hosting - 限时3天` |
| 校验提示 | 为空或超长时给予友好错误提示                  |

### 2.3 标签（Tags）

- 改为**标签输入组件**（输入后回车生成标签样式）
- 支持一键删除已添加标签
- 预设热门标签快捷添加：
    - `kvm`、`cheap`、`us`、`annual`、`unlimited`、`ssd`、`dDoS保护`

### 2.4 Markdown 内容编辑

| 优化点 | 说明 |
|--------|------|
| 富文本工具栏 | 提供加粗、斜体、列表、链接、图片、标题等快捷按钮 |
| 编辑模式切换 | 支持 `源码` / `富文本` / `实时预览` 三种模式 |
| 字数统计 | 显示当前字数，建议 200~500 字 |
| 内容长度提示 | 过短或过长时给出建议提示 |

---

## 三、预览区优化

### 3.1 多设备预览
在预览区顶部增加设备切换按钮：

| 设备 | 图标示意 |
|------|----------|
| 桌面 | 🖥️ |
| 平板 | 📱 |
| 手机 | 📲 |

> 点击后预览区自适应对应设备宽度，模拟前台真实展示效果。

### 3.2 预览环境提示
在预览区顶部显示提示文字：

> 🔍 *以下为优惠发布后的前台展示效果，仅供预览参考。*

### 3.3 关键信息高亮
在预览中自动高亮显示以下字段，便于核验：
- 优惠标题
- 折扣力度/价格
- 有效期
- 优惠码（如有）
---

## 四、交互与反馈优化

### 4.1 实时校验
- 标题：必填，不超过 60 字符
- 分类：至少选择一个
- Markdown：不能为空
- 校验时机：字段失去焦点时触发
- 错误提示：使用友好、明确的文案

### 4.2 草稿保存
- 增加 **“保存草稿”** 按钮（与“发布”并列）
- 草稿自动保存在本地或服务端
- 支持断点续填，避免内容丢失

### 4.3 发布流程反馈

| 阶段 | 交互反馈 |
|------|----------|
| 点击发布 | 按钮变为 `发布中...`，禁用重复点击 |
| 发布成功 | 弹出成功提示 ✅，并提供两个操作按钮：`查看已发布优惠` / `继续创建` |
| 发布失败 | 弹出错误提示 ❌，显示具体失败原因，保留已填内容 |

## 五、其他增强点

### 5.1 有效期设置
- 增加 **生效时间** 和 **截止时间** 日期选择器
- 支持快捷选项：`1天后`、`3天后`、`7天后`、`永久有效`

### 5.2 优惠码字段
- 独立输入框，用于填写优惠码
- 提供开关选项：`是否在页面公开显示优惠码`

### 5.3 SEO 预览（进阶）
- 在预览区下方增加 SEO 标题和描述预览
- 帮助用户优化搜索引擎展示效果
